# A_C++

## Platform

Codeforces

## Problem Link

[A. C+=](https://codeforces.com/problemset/problem/1368/A)

## Problem Statement

You are given three integers:

* `a`
* `b`
* `n`

You may repeatedly perform the following operation:

* Replace the **smaller number** with the sum of both numbers.

More precisely, in one operation:

```text
a = a + b
```

or

```text
b = a + b
```

depending on which one is smaller.

Your goal is to determine the **minimum number of operations required** until **either `a` or `b` becomes strictly greater than `n`**.

## Input Format

* The first line contains an integer `t` — the number of test cases.
* Each test case contains three integers:

```text
a b n
```

## Output Format

For each test case, print a single integer — the **minimum number of operations required**.

## Constraints

* `1 ≤ t ≤ 1000`
* `1 ≤ a, b ≤ n ≤ 10^9`

## Approach

Greedy Simulation with Fibonacci-like Growth

## Intuition

Each operation replaces the smaller number with the **sum of both numbers**.

This produces a sequence similar to the **Fibonacci sequence**, where values grow rapidly.

Example pattern:

```
a, b
b, a+b
a+b, a+2b
a+2b, 2a+3b
...
```

Thus, repeatedly adding the two numbers will eventually exceed `n`.

To minimize operations:

* Always ensure the smaller value is updated.
* Swap `a` and `b` if necessary so `a ≤ b`.

Then repeatedly:

```
new = a + b
a = b
b = new
```

Continue until `b > n`.

## Algorithm Steps

* Read integer `t`.
* For each test case:

  * Read `a`, `b`, `n`.
  * If `a > b`, swap them.
  * Initialize `count = 0`.
  * While `b ≤ n`:

    * Compute `temp = a + b`.
    * Update:

      ```
      a = b
      b = temp
      ```
    * Increment `count`.
  * Print `count`.

## Complexity Analysis

### Time Complexity

```
O(log n)
```

The values grow similarly to Fibonacci numbers, so they exceed `n` in logarithmic steps.

### Space Complexity

```
O(1)
```

Only a few integer variables are used.

## Solution Explanation

The program simulates the process of repeatedly adding the two numbers.

Before starting the loop, it ensures:

```
a ≤ b
```

This guarantees the smaller value is always replaced with the sum.

During each iteration:

```
new_value = a + b
```

The numbers shift forward like a Fibonacci sequence.

The process continues until the larger number becomes greater than `n`.

The number of iterations equals the minimum operations required.

## Full Solution

### C++ Implementation

```cpp
#include<iostream>
using namespace std;

int main() {

    // Number of test cases
    int t, a, b, n;
    cin >> t;

    while(t--) {

        // Read values
        cin >> a >> b >> n;

        // Ensure a ≤ b
        if(a > b) {
            int temp = a;
            a = b;
            b = temp;
        }

        // Operation counter
        int count = 0;

        // Continue until one number exceeds n
        while(b <= n) {

            // Compute next value
            int temp = a + b;

            // Update values similar to Fibonacci sequence
            a = b;
            b = temp;

            // Increment operation count
            count++;
        }

        // Output result
        cout << count << '\n';
    }

    return 0;
}
```
